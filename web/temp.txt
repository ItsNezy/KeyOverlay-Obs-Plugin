1. Overview
KeyOverlay adalah web-based keyboard overlay yang dirancang untuk digunakan sebagai Browser Source di OBS Studio. Tool ini memungkinkan streamer menampilkan visual keyboard real-time dengan tampilan yang fully customizable — mulai dari layout, warna, ukuran, hingga animasi keypress.
Dua target user utama:
Casual streamer — tinggal buka di browser, drag-and-drop lewat UI editor, langsung jalan di OBS.
Power user / developer — bisa oprek CSS sendiri, export/import config JSON, atau bahkan extend fitur via custom script.
2. Goals & Non-Goals
Goals
Keyboard overlay jalan sebagai Browser Source di OBS tanpa instalasi apapun.
Customizable via GUI editor (no-code) untuk casual user.
Customizable via CSS/JSON untuk power user.
Ringan dan performa tinggi — HTML/CSS/JS murni, zero framework besar.
Keypress terdeteksi real-time dengan visual feedback (highlight, animasi, dll).
Support berbagai layout keyboard: QWERTY, AZERTY, custom.
Config bisa disimpan dan di-share antar user (export/import JSON).
Non-Goals
Tidak ada backend server — semua client-side.
Tidak support Linux native keylogging (browser limitation).
Tidak ada cloud sync atau akun user.
Tidak perlu support mobile atau touch input.
3. User Stories
Casual Streamer
Sebagai streamer, saya ingin membuka URL di OBS Browser Source dan langsung lihat keyboard overlay tanpa setup ribet.
Sebagai streamer, saya ingin ganti warna dan font via panel editor tanpa perlu tau CSS.
Sebagai streamer, saya ingin pilih preset tema yang sudah jadi (dark, light, gaming, aesthetic, dsb).
Sebagai streamer, saya ingin key yang ditekan langsung highlight dengan animasi smooth.
Power User / Developer
Sebagai developer, saya ingin inject custom CSS langsung ke overlay tanpa edit source code.
Sebagai developer, saya ingin export konfigurasi ke JSON dan import lagi di browser lain.
Sebagai developer, saya ingin definisikan layout keyboard custom (misal: 60%, TKL, numpad only).
Sebagai developer, saya ingin set highlight color per-key group (modifier, alphanumeric, function keys, dll).
4. Arsitektur Sistem
Stack
Frontend
HTML5, CSS3 (Custom Properties), Vanilla JavaScript (ES6+)
Input Detection
KeyboardEvent API (browser-native, via OBS Browser Source)
Storage
localStorage — menyimpan config user secara lokal
Distribusi
Single HTML file — bisa dihosting di GitHub Pages, Netlify, dsb
Cara Kerja di OBS
User menambahkan Browser Source di OBS dengan URL dari overlay. OBS akan render halaman tersebut menggunakan Chromium engine — sehingga keyboard events bisa terdeteksi selama Browser Source dalam kondisi fokus (atau menggunakan OBS global key hook jika diperlukan).
Flow sederhana:
User buka URL overlay di browser untuk konfigurasi.
Setelah puas, copy URL beserta query params config (atau gunakan localStorage) ke OBS Browser Source.
OBS render overlay. Keypress terdeteksi dan divisualisasikan real-time.

6. UI & UX Spec
6.1 Mode Overlay (OBS View)
Background: transparent (rgba(0,0,0,0)).
Keyboard container: center horizontal, posisi bisa dikonfigurasi (bottom, custom x/y).
Key default: rounded rectangle, warna sesuai tema.
Key pressed: warna berubah + transisi smooth (default 100ms ease).
Tidak ada UI editor — clean, hanya keyboard.
6.2 Mode Editor (Config View)
Diakses dengan menambahkan ?editor=true ke URL, atau tombol khusus yang hilang saat di OBS.
Panel kiri: preview keyboard live.
Panel kanan: kontrol customization (color pickers, sliders, dropdowns).
Tab: Themes | Layout | Keys | Advanced CSS | Export/Import.
Tombol 'Copy OBS URL' — generate URL dengan config ter-encode.
7. Data Model
Config Schema (JSON)

{
  "theme": "dark",
  "layout": "full",
  "scale": 1.0,
  "position": { "x": "center", "y": "bottom" },
  "colors": {

    "keyText": "#ffffff",
    "keyBorder": "#444444",
    "keyPressed": "#3B82F6",
    "keyPressedText": "#ffffff",

  },
  "font": {
    "family": "Inter, sans-serif",
    "size": 12
  },
  "animation": {
    "type": "fade",
    "duration": 100
  },
  "customCSS": "",
  "perKeyColors": {}
}

File Structure
keyoverlay/
├── index.html          # Entry point, overlay + editor UI
├── css/
│   ├── base.css        # Reset, CSS variables, base styles
│   ├── keyboard.css    # Key layout & rendering
│   ├── themes.css      # Preset theme definitions
│   └── editor.css      # Editor panel styles
├── js/
│   ├── main.js         # Init, mode detection (overlay vs editor)
│   ├── keyboard.js     # Render keyboard dari layout JSON
│   ├── keypress.js     # Keydown/keyup event handler
│   ├── config.js       # Load/save config ke localStorage
│   ├── editor.js       # Editor panel logic
│   └── export.js       # Export/import JSON
└── layouts/
    ├── full.json        # Full QWERTY 104-key layout
    ├── tkl.json         # Tenkeyless 87-key layout
    ├── 60.json          # 60% layout
    └── numpad.json      # Numpad only

9. Constraints & Assumptions
OBS Browser Source menggunakan Chromium — semua fitur harus kompatibel dengan Chrome/Chromium terbaru.
Keyboard events HANYA terdeteksi jika Browser Source dalam kondisi fokus di OBS, atau user menggunakan OBS interaction mode.
Zero dependency eksternal — tidak ada npm, tidak ada CDN runtime (kecuali font dari Google Fonts opsional).
Total ukuran file harus di bawah 500KB untuk load time cepat.
Tidak ada server — semua logic berjalan di browser.

11. Out of Scope
Global keylogging di luar browser window — ini limitasi browser, bukan bug.
Support macOS media keys atau special keys OS-level.
Fitur multiplayer / sharing real-time.
Recording atau replay keypress history.
Mobile / tablet support.
